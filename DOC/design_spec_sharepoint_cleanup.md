# SharePoint Version Cleanup System 設計書

## 1. 目的

Power Automate から HTTP 経由で Python プログラムを呼び出し、 SharePoint
のファイルバージョン履歴を再帰的に削除する。

------------------------------------------------------------------------

## 2. 全体構成

Power Automate ↓ (HTTP POST) Python API (FastAPI) ↓ SharePoint REST API

------------------------------------------------------------------------

## 3. 設計思想

### 3.1 Power Automate 側の役割

-   スケジューリング
-   トリガー管理
-   API呼び出しのみ

👉 重い処理は持たせない

------------------------------------------------------------------------

### 3.2 Python 側の役割

-   再帰的フォルダ探索
-   ファイル一覧取得
-   バージョン取得
-   古いバージョン削除
-   ログ管理

👉 ロジックはすべてPython側に集約

------------------------------------------------------------------------

### 3.3 なぜこの構成か

#### ❌ Power Automate単体の問題

-   ループ制限
-   再帰が苦手
-   フローが複雑化

#### ✅ Python化のメリット

-   再帰が自然に書ける
-   制限がほぼ無い
-   デバッグしやすい

------------------------------------------------------------------------

## 4. 処理フロー

1.  Power Automate が HTTP POST
2.  Python API がリクエスト受信
3.  cleanup処理を開始
4.  フォルダを再帰的に走査
5.  ファイル取得
6.  バージョン取得
7.  古いバージョン削除
8.  結果を返却

------------------------------------------------------------------------

## 5. API設計

### POST /cleanup

同期処理

### POST /cleanup/background

非同期処理

### GET /health

疎通確認

------------------------------------------------------------------------

## 6. リクエスト例

{ "root_folder_server_relative_url": "...", "days_to_keep": 30,
"dry_run": true }

------------------------------------------------------------------------

## 7. セキュリティ

-   APIキー認証
-   HTTPS必須

------------------------------------------------------------------------

## 8. 結論

Power Automate = トリガー\
Python = 実処理

👉 責務分離が重要
